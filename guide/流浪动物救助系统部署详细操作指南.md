### 3.3 修改 Dockerfile 镜像源

**问题**: Dockerfile 使用了国内镜像源 `docker.m.daocloud.io`，应该改为官方源

**查看原始 Dockerfile**:
```bash
cat backend/Dockerfile
```

**原始内容**:
```
FROM docker.m.daocloud.io/library/node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["node", "app.js"]
```

**修改后**:
```
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["node", "app.js"]
```

**修改命令**:
```bash
# 将镜像源从 daocloud 改为官方源
sed -i 's/docker\.m\.daocloud\.io\/library\//docker.io\//' backend/Dockerfile
```

**验证修改**:
```bash

```

**实际输出**:

### 5.1 创建数据库用户

**问题**: Docker Compose 配置中定义了 `backend_user` 用户，但数据库容器中没有创建这个用户

```bash
# 在数据库中创建 backend_user 用户
docker exec animal_db mysql -uroot -proot123 -h 127.0.0.1 animal_rescue -e "
CREATE USER IF NOT EXISTS 'backend_user'@'%' IDENTIFIED BY 'backend123';
GRANT ALL PRIVILEGES ON animal_rescue.* TO 'backend_user'@'%';
FLUSH PRIVILEGES;
"
```

**实际输出**: 无输出（执行成功）

### 5.3 修复数据库中的 URL

**问题**: 数据库中的图片 URL 指向错误的端口 :3001，应该改为 :8081

**修改前**: URL 包含 `:3001` 端口

**修改后**: URL 包含 `:8081` 端口

```bash
UPDATE RescueStations SET images = REPLACE(images, ':3001', ':8081') WHERE images LIKE '%:3001%';
UPDATE animals SET photos = REPLACE(photos, ':3001', ':8081') WHERE photos LIKE '%:3001%';
UPDATE news SET cover_image = REPLACE(cover_image, ':3001', ':8081') WHERE cover_image LIKE '%:3001%';
UPDATE users SET avatar = REPLACE(avatar, ':3001', ':8081') WHERE avatar LIKE '%:3001%';
"
```

**验证修改**:

## 🚨 第十步：遇到的问题和解决方案

### 10.1 问题1：前端无法连接后端 API

**错误信息**: `net::ERR_CONNECTION_REFUSED`

**原因**: 前端构建文件中的 API 地址配置错误

**解决方案**:
1. 修改 `frontend/.env.production` 文件，将 `VITE_API_BASE_URL` 从 `http://localhost:8081/api` 改为 `/api`
2. 重新构建前端项目：`npm run build`
3. 复制构建文件到 nginx 目录：`cp -r dist/* /root/animial_rescue/frontend-dist/`
4. 重启前端容器：`docker restart animal_web`

### 10.2 问题2：数据库连接失败

**错误信息**: `Can't connect to MySQL server on 'animal-mysql'`

**原因**: 后端配置文件中的 `DB_HOST=animal-mysql` 与 Docker Compose 中的 `db` 不匹配

**解决方案**:
1. 修改 `backend/.env` 文件，将 `DB_HOST=animal-mysql` 改为 `DB_HOST=db`
2. 重启后端容器：`docker restart animal_backend`

### 10.3 问题3：管理员登录失败

**错误信息**: `Invalid password`

**原因**: 数据库中的管理员密码哈希不正确

**解决方案**:


### 10.4 问题4：数据库用户不存在

**错误信息**: `Access denied for user 'backend_user'@'%'`

**原因**: 数据库容器中没有创建 `backend_user` 用户

**解决方案**:
1. 在数据库中创建用户：
   ```bash
   docker exec animal_db mysql -uroot -proot123 -h 127.0.0.1 animal_rescue -e "CREATE USER IF NOT EXISTS 'backend_user'@'%' IDENTIFIED BY 'backend123'; GRANT ALL PRIVILEGES ON animal_rescue.* TO 'backend_user'@'%'; FLUSH PRIVILEGES;"
   ```
2. 重启后端容器：`docker restart animal_backend`
