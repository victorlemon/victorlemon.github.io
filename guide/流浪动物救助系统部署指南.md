本指南将详细介绍如何从零开始部署流浪动物救助系统，适用于新手用户。文档包含了每一个操作步骤、输入的命令以及详细的解释。

## 🔧 第一步：环境准备

### 1.1 检查 Docker 环境

在开始部署之前，我们需要确保服务器上已经安装并正确配置了 Docker 和 Docker Compose。

```bash
# 检查 Docker 是否安装
docker --version

# 检查 Docker Compose 是否安装
docker compose version 2>/dev/null || docker-compose --version 2>/dev/null
```

**命令解释：**

```text
docker --version - 检查 Docker 版本，确认 Docker 已安装
docker compose version - 检查 Docker Compose 版本（新版本）
docker-compose --version - 检查 Docker Compose 版本（旧版本）
```

**预期输出：**
```
Docker version 29.6.1, build 8900f1d
Docker Compose version v5.2.0
```

### 1.2 检查端口占用

我们需要确保部署所需的端口没有被其他服务占用。

```bash
# 检查常用端口是否被占用
ss -tlnp | grep -E ':(3307|3001|8081|80|3000|8080)\s'
```

**命令解释：**

```text
ss -tlnp - 显示 TCP 端口监听状态
grep -E ':(3307|3001|8081|80|3000|8080)\s' - 过滤出我们需要关注的端口
```

**预期输出：**
```
# 如果端口没有被占用，不会有任何输出
# 如果端口被占用，会显示类似这样的信息：
# LISTEN 0 511 *:8081 *:*
```

### 1.3 检查现有容器状态

```bash
# 查看所有容器状态
docker ps -a --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

**命令解释：**

```text
docker ps -a - 查看所有容器（包括停止的）
--format "table {{.Names}}\t{{.Status}}\t{{.Ports}}" - 以表格格式显示容器名称、状态和端口映射
```

**预期输出：**
```
NAMES           STATUS          PORTS
# 如果没有容器，会显示空表头
```

### 3.2 修复后端数据库连接配置

**问题：** 后端配置文件中的 `DB_HOST=animal-mysql`，但在 Docker Compose 中数据库服务名为 `db`，这会导致连接失败。

```bash
# 修改后端配置文件，将数据库主机名改为正确的服务名
sed -i 's/DB_HOST=animal-mysql/DB_HOST=db/' backend/.env
```

**命令解释：**

```text
sed -i 's/old_string/new_string/' filename - 在文件中替换字符串
这里将 animal-mysql 改为 db，因为 Docker Compose 中数据库服务的名称是 db
```

**验证修改：**
```bash
cat backend/.env
```

**预期输出：**
```
# 数据库配置（给 Docker 容器用）
DB_NAME=animal_rescue
DB_HOST=db  # 这里已经修改为 db
DB_PORT=3306
DB_USER=backend_user
DB_PASSWORD=backend123
# ... 其他配置
```

### 3.3 前端环境配置

**问题：** 前端需要正确的 API 地址配置，使用相对路径 `/api` 而不是绝对路径。

```bash
# 修改前端环境配置文件
cat > frontend/.env.production << 'EOF'
# 后端接口配置
# 生产环境：使用相对路径，通过 Nginx 代理
VITE_API_BASE_URL=/api

# 图片服务器地址
VITE_SERVER_URL=

# 腾讯地图 API Key (需用户自行申请)
VITE_TMAP_KEY=YOUR_TENCENT_MAP_KEY_HERE
EOF
```

**命令解释：**

```text
cat > filename << 'EOF' - 使用 here document 写入文件
VITE_API_BASE_URL=/api - 使用相对路径，Nginx 会将请求代理到后端
VITE_SERVER_URL= - 清空图片服务器地址，使用相对路径
```

**验证配置：**
```bash
cat frontend/.env.production
```

### 3.4 修复 Dockerfile 镜像源

**问题：** Dockerfile 中使用了国内的镜像源 `docker.m.daocloud.io`，可能导致下载缓慢。

```bash

```

**修改 Dockerfile：**
```bash
# 将镜像源从 daocloud 改为官方源

```


## 🔧 第五步：问题排查和解决

### 5.1 数据库用户创建问题

**问题：** Docker Compose 配置中定义了 `backend_user` 用户，但数据库容器中没有创建这个用户。

```bash
# 在数据库中创建 backend_user 用户
docker exec animal_db mysql -uroot -proot123 -h 127.0.0.1 animal_rescue -e "
CREATE USER IF NOT EXISTS 'backend_user'@'%' IDENTIFIED BY 'backend123';
GRANT ALL PRIVILEGES ON animal_rescue.* TO 'backend_user'@'%';
FLUSH PRIVILEGES;
"
```

**命令解释：**

```text
docker exec animal_db mysql -uroot -proot123 -h 127.0.0.1 animal_rescue -e "SQL命令" - 在数据库容器中执行 SQL 命令
CREATE USER IF NOT EXISTS 'backend_user'@'%' IDENTIFIED BY 'backend123' - 创建用户
GRANT ALL PRIVILEGES ON animal_rescue.* TO 'backend_user'@'%' - 授权用户访问数据库
FLUSH PRIVILEGES - 刷新权限
```

### 5.2 管理员密码修复

**问题：** 数据库中的管理员密码哈希不正确，导致登录失败。

```bash
# 使用正确的密码哈希更新管理员密码
docker exec animal_db mysql -uroot -proot123 -h 127.0.0.1 animal_rescue -e "
UPDATE users SET password='\$2a\$12\$qSQo1zbmgMxPEOcUYwvkYedPFZ9TdbYFukTsqdNzgU.694KG/KOLe' WHERE username='admin';
"
```

**命令解释：**

```text
这里使用 bcrypt 生成的正确哈希值，对应密码 "admin123"
注意：\$2a\$12\$ 是 bcrypt 哈希的前缀，需要转义
```

### 5.3 数据库 URL 修复

**问题：** 数据库中的图片 URL 指向错误的端口。

```bash
# 修复数据库中的 URL，将 :3001 改为 :8081
docker exec animal_db mysql -uroot -proot123 -h 127.0.0.1 animal_rescue -e "
UPDATE RescueStations SET cover_image = REPLACE(cover_image, ':3001', ':8081') WHERE cover_image LIKE '%:3001%';
UPDATE RescueStations SET images = REPLACE(images, ':3001', ':8081') WHERE images LIKE '%:3001%';
UPDATE animals SET photos = REPLACE(photos, ':3001', ':8081') WHERE photos LIKE '%:3001%';
UPDATE news SET cover_image = REPLACE(cover_image, ':3001', ':8081') WHERE cover_image LIKE '%:3001%';
UPDATE users SET avatar = REPLACE(avatar, ':3001', ':8081') WHERE avatar LIKE '%:3001%';
"
```

**命令解释：**

```text
使用 REPLACE 函数替换 URL 中的端口号
确保所有图片 URL 都指向正确的 Nginx 代理端口 8081
```

### 5.4 重启服务

```bash
# 重启后端服务
docker restart animal_backend

# 等待后端重启
sleep 5

# 重启前端服务
docker restart animal_web

# 等待前端重启
sleep 5
```

**命令解释：**

```text
docker restart container_name - 重启指定容器
sleep 5 - 等待服务重启完成
```

## 🚨 第九步：常见问题排查

### 9.1 端口被占用

**问题：** 启动容器时提示端口被占用

**解决方案：**
```bash
# 查看端口占用情况
ss -tlnp | grep -E ':(3307|3001|8081)\s'

# 终止占用端口的进程
sudo kill -9 <PID>
```

### 9.2 数据库连接失败

**问题：** 后端日志显示数据库连接错误

**解决方案：**
```bash
# 检查后端日志
docker logs animal_backend --tail 20

# 检查数据库容器状态
docker logs animal_db --tail 20

# 确保数据库用户已创建
docker exec animal_db mysql -uroot -proot123 -h 127.0.0.1 animal_rescue -e "SELECT user,host FROM mysql.user WHERE user='backend_user';"
```

### 9.3 前端无法访问 API

**问题：** 前端显示网络错误，无法连接到后端 API

**解决方案：**
```bash
# 检查前端构建文件中的 API 地址
grep -o 'baseURL:"[^"]*"' /root/animial_rescue/frontend-dist/assets/index-*.js

# 如果地址不正确，重新构建前端
cd /root/animial_rescue/frontend
npm run build
cp -r dist/* /root/animial_rescue/frontend-dist/
docker restart animal_web
```

### 9.4 容器启动失败

**问题：** 容器启动失败或反复重启

**解决方案：**
```bash

docker-compose -f docker-compose.local logs

# 重新启动所有服务
```
